# Documentation Maintenance Checklist

- Freshness source captured:
  - command or report path
- Target docs identified:
  - paths
  - `staleness_level`
  - review reference problems
- Maintenance class chosen for each doc:
  - substantive review/update
  - review evidence refresh
  - structure/metadata repair
  - governance escalation
- `review mark` suitability checked explicitly
- Rule/routing escalation checked where stale signal looked suspicious
- Validation commands prepared
- Follow-up maintainer skill identified when the work is no longer document-focused
